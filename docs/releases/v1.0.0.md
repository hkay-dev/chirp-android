# Chirp 1.0.0

Chirp 1.0.0 is the first public APK release. It records voice notes, transcribes speech on the
device, and gives you a few ways to clean up or reuse the result.

## What works today

### Recording and playback

- Record in M4A, MP3, or WAV from the main app.
- Keep recording through the foreground service while the app is in the background.
- Pause, resume, save, discard, search, and play recordings.
- Import an existing audio file and send it through the same transcription flow.
- Recover interrupted recording sessions when enough audio was saved.

### Offline transcription

- Download the NVIDIA Parakeet TDT 0.6B V2 INT8 model from the transcription settings screen.
- Transcribe recordings locally with sherpa-onnx and ONNX Runtime.
- Queue transcription work in the background.
- Apply custom word replacements to recurring names, terms, or abbreviations.

The model download needs an internet connection. Once the model is installed, speech-to-text runs
on the device.

### Processing Studio

- Read and edit transcripts.
- Generate cleaned text, titles, summaries, meeting notes, and action items.
- Ask questions about a transcript in the chat tab.
- Keep structured processing results with the recording.

AI processing is optional. It needs an internet connection and your own API key for Google Gemini,
OpenAI, Anthropic, Groq, or Cerebras. Text sent for AI processing goes to the provider you choose.
Chirp doesn't bundle an API key.

### Voice input around Android

- Use Chirp Voice as an Android keyboard and insert local transcription into the current text field.
- Use Chirp as an Android speech recognition service from compatible apps and keyboards.
- Start or stop recording from the home-screen widget.

### Organization and export

- Organize recordings with profiles and tags.
- Give profiles their own transcription, processing, audio, and export defaults.
- Export transcripts to an Obsidian vault as Markdown.
- Share transcript text or audio from the app.

## Requirements and current limits

- Android 16 is required because this build targets and supports API 36 only.
- The speech model must be downloaded before recording or local transcription can start.
- Every hands-on test so far has been on a Galaxy S25 Ultra. That's the only Android device
  available for testing, so feedback from other devices would be very useful. The APK also
  packages the native libraries supplied for 32-bit Arm, x86, and x86_64.
- Long-session recovery, low-storage handling, background finalization, keyboard handoff, and queue
  recovery have automated coverage. The full 60+ minute and multi-OEM physical-device checklist is
  still ongoing.
- AI cleanup isn't local. Only speech-to-text runs fully offline after the model download.
- Gboard doesn't expose the compatible external speech recognition picker used by Chirp. SwiftKey
  supports that flow, and Chirp Voice works as its own keyboard.

## Verification

The release build runs through detekt, Android lint, module unit tests, database and migration test
compilation, recording and transcription recovery tests, keyboard lifecycle tests, and release APK
signature verification.
